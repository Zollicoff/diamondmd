# Philosophy

## The three things DiamondMD commits to

1. **Markdown files, flat on disk.** If you uninstall DiamondMD tomorrow, your notes still exist. Every feature we add has to preserve this. No proprietary format, no database-of-record, no lock-in.

2. **Git as the versioning layer.** Not bolted on — load-bearing. Sync is `git push`. History is `git log`. Drafts are branches. This is simpler, freer, and more powerful than any proprietary sync protocol. See [[Git Versioning]].

3. **Web-first.** Browsers are the universal runtime. Works on desktop, tablet, phone — same URL, no per-device install. No Electron.

## What we're not trying to be

- **Obsidian.** We're a subset. Their plugin ecosystem is their moat; we don't try to replicate it.
- **Notion.** Databases, tables, collaborative real-time editing — different product.
- **A wiki.** Wikis are multi-user by design; DiamondMD is single-user by default.
- **A publishing platform.** Publishing is [[Versioning|v0.4]]; even then it's a feature, not the core.

## Why open source

DiamondMD only works if the code can outlive the maintainer. Every feature is a commitment to keep the file format, git layout, and wikilink resolution stable. Open source is the credible way to make that commitment.

MIT license. Fork it, sell it, embed it — we don't care. The only thing that'd upset us is someone adding DRM to a fork.

## Why the name

Diamonds are crystalline carbon. Same atoms as [[Versioning|Obsidian]] (sort of), rearranged under pressure into something more durable. Also the repo name wasn't taken.
